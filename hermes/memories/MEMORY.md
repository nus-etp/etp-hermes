Hermes venv at ~/.hermes/hermes-agent/venv/ ships without pip. To install packages into it, use `uv pip install <pkg> --python ~/.hermes/hermes-agent/venv/bin/python` — NOT `uvx --python ... pip install` (uvx installs into a temp ephemeral env, not the target venv). This is the common root cause for "No adapter available for telegram/discord/slack" errors in gateway logs.

Synthesis pipeline (Layer 3): reads `signals/updates/<date>.md` and `signals/agent/<date>.md`, writes/updates per-company `signals/briefs/<slug>/LIVING_BRIEF.md`. For existing briefs, dedup by URL against both Recent and Older signals sections before prepending. Cap is 20 bullets in Recent signals; older ones move to Older signals. No-write check compares merged content byte-for-byte with on-disk file. Slugs derived from company name: lowercase, `[^a-z0-9]` -> `-`, trim. Source-short for link brackets should be publication name (e.g., "The Business Times", "Devdiscourse", "Patsnap").

Jina pre-extracted items for per-company html_scrape pages are often low-quality — they extract navigation elements (Company, Resources, get in touch) and image links (cdn.prod.website-files.com), not actual news articles. The `pre_extracted: true` flag doesn't guarantee a useful item. Additional filtering is needed: skip items with short headlines (< 15 chars), CDN image URLs, or nav-text content. Also, many per-company html_scrape pages (Craft Health, Aurora Food, SINGROW) have content that is 2+ years old, so the 14-day recency filter catches nothing.
